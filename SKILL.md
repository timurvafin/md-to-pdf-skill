# Скилл: конвертация Markdown → PDF

## Процесс

### Шаг 1: Определить входной файл

- Пользователь указывает путь к Markdown-файлу или просит конвертировать контент из разговора.
- Если файл не указан — спросить у пользователя путь к исходному файлу.
- Если нужно конвертировать inline-контент, сначала записать его во временный `.md` файл.

### Шаг 2: Выбрать тему

Тема выбирается по типу контента или по явной просьбе пользователя:

| Тип контента | Тема | Файл |
|---|---|---|
| Техническая документация, код | `modern` | `theme-modern.css` |
| Деловые документы, отчёты | `professional` | `theme-professional.css` |
| Научные статьи, академические тексты | `academic` | `theme-academic.css` |
| Тёмный режим / чтение с экрана | `dark` | `theme-dark.css` |
| Чистый минималистичный вывод | `minimal` | `theme-minimal.css` |
| По умолчанию (если неясно) | `professional` | `theme-professional.css` |

Если тема не указана явно — автоопределение по контенту:
- Много блоков кода → `modern`
- Формальный язык, цитаты, сноски → `academic`
- Во всех остальных случаях → `professional`

### Шаг 3: Запустить конвертацию

Использовать скрипт конвертации:

```bash
bash <SKILL_DIR>/scripts/convert.sh -i INPUT.md -o OUTPUT.pdf -t ТЕМА [-l]
```

> `<SKILL_DIR>` — директория, в которую установлен скилл (например, `~/.claude/skills/md-to-pdf`).

Аргументы:
- `-i` — входной Markdown-файл (обязательно)
- `-o` — выходной PDF-файл (по умолчанию: то же имя с расширением `.pdf`)
- `-t` — название темы без префикса `theme-` (по умолчанию: `professional`)
- `-l` — landscape ориентация (по умолчанию: portrait)

### Шаг 4: Сообщить результат

- Показать путь к созданному PDF.
- Если конвертация упала — прочитать вывод ошибки и устранить проблему.

## Расположение CSS-тем

Все темы находятся в: `<SKILL_DIR>/assets/`

## Примечания

- Пайплайн: pandoc → HTML+CSS → Chrome headless → PDF
- Все темы поддерживают кириллицу (PT Sans, PT Serif, Fira Code)
- Для корректного отображения изображений они должны быть в той же папке, что и исходный файл, или использовать абсолютные пути
